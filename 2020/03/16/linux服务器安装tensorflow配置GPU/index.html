环境：linux+Anaconda3+python3.5+CUDA9.0+cudnn7.0

###1.conda 创建虚拟环境(python3.6+tensorflow1.6)

conda create -n tensorflow1.6 python=3.6

安装tensorflow1.6(cpu版本)

pip install tensorflow==1.6

####2.2 CUDA下载及安装

查看linux内核，对于isyslab服务器g-1-4，g-1-5是centos 7.3 redhat，g-1-6是6.5；

确认CUDA与cudnn配对版本,这里选择tensorflow1.6对应的cuDNN7，CUDA9；（最新的tensorflow需要使用CUDA10.1）

下载CUDA9.0如下：

输入sh cuda_9.0.176_384.81_linux-run进行安装

然后输入nvcc -V，如果显示CUDA版本信息代表正常。

####2.3 cuDNN下载及配置

到nvidia网站下载与CUDA9.0对应的cuDNN

教程里是下载得到tgz文件直接解压，我用mac下载是.arj文件，服务器上没有arj解压（或者我不知道），最后是在mac上先解压cudnn-9.0-linux-x64-v7.6.5.32.arj得到cudnn-9.0-linux-x64-v7.6.5.32，这个文件还是个压缩文件，猜测是tgz文件，于是tar -zxvf cudnn-9.0-linux-x64-v7.6.5.32,还真的解压出了，把解压缩得到的文件上传到服务器，拷贝到相应位置

4.小结

1.在isyslab的gpu节点上，各个gpu节点的配置是不同的：例如g-1-4，g-1-5是centos 7.3 redhat，g-1-6是6.5。一开始在g-1-6安装tensorflow1.6报错”libc.so.6: version ‘GLIBC_2.14’ not found”，查看系统glibc支持的版本：strings /lib64/libc.so.6 |grep GLIBC_,发现确实没有，但是在g-1-4和g-1-5上都有，所以在这两个节点上安装成功。
